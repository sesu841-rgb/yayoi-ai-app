# 人生構造レポート LPサイト構造設計

## 1. LPページ構造
サイトは**「シングルページアプリケーション（SPA）」**をベースとした、スクロールに特化した縦長1ページ構成とします。
ユーザーが離脱しにくいよう、決済からバックエンドのシステム連動までシームレスに行える構造をとります。

- **メイン画面**: LP本体（販売ページ）
- **別タブ/モーダル**: 特商法・プライバシーポリシー・利用規約
- **決済後ページ**: 入力フォーム画面 → 生成ローディング画面 → PDFダウンロード画面

---

## 2. セクション構成
ご指定の順序に従い、スマホでのスクロールリズムを考慮して配置します。

1. **ヒーローセクション（ファーストビュー）**: 画像を使わず、白背景に鋭いテキストのみ。「自分のことだ」と思わせるキャッチコピー配置。
2. **共感セクション**: スマホの「1画面に1メッセージ」の余白を取り、箇条書きで違和感を言語化。
3. **構造説明**: 「努力不足ではない」という核心のパラダイムシフトを大きな文字で提示。
4. **ストーリー**: 開発者の「年収1000万→洗脳→全損」の転落を時系列で静かに語る（テキスト中心）。
5. **レポート説明**: 本商品で明らかになることを、チェックリスト形式で端的に表示。
6. **レポート例**: サンプル構成（0〜7章）。長くなる場合はアコーディオン（タップで開閉）を採用し、スマホでの縦幅を取りすぎないようにする。
7. **利用者の声**: カルーセル（横スワイプ）またはカード型UIで6名分のリアルな声を配置。
8. **診断質問**: ユーザー自身が心の中でチェックできる「チェックボックス風」のUIを採用し、自己認識を促す。
9. **価格**: 「個別解析レベル」という価値と「2980円」のコントラストを静かに、しかし力強く見せる。
10. **CTA**: レポートを受け取るボタンへの最終導線。

## 3. UI設計
スマートフォンでの閲覧体験を最優先とした、「静かで鋭く、知的な」デザインシステム。

- **カラーパレット**
  - **背景色**: 真っ白（#FFFFFF）〜かすかなオフホワイトで静寂を表現。
  - **テキスト色**: 漆黒〜墨色（#222222）でハッキリ読みやすく。
  - **アクセント色**: 深い藍色（ネイビー）やチャコールグレー。煽り要素となる赤や太い黄色マーカーは排除し、知的な「孔明の切れ味」を演出。
- **タイポグラフィ**
  - 洗練された明朝体（Noto Serif JP等）、またはスマートなモダンゴシック体。
  - 行間（line-height: 1.8~2.0）や字間を広めに取り、「読む」というより「心にしみこむ」リズムを作る。
- **レイアウト・スマホ最適化**
  - 左右に十分な余白（padding: 24pxなど）を取り、文章が中央に寄るように設計。
  - セクション間の余白（margin）を非常に大きくし、改行ごとに一呼吸おけるスペースデザインにする。
- **CTAボタン**
  - ボタンは立体感を抑えたフラットデザイン、もしくは微かなシャドウ。ボタンテキストは「人生構造レポートを受け取る - 2980円」。

## 4. 入力フォーム
決済完了後に遷移する、レポート生成のためのデータ入力フロー。
40代ユーザーの認知負荷を下げるため、「1画面1ステップ（ウィザード形式）」を採用します。

- **STEP 1：基本情報**
  - 現在の年齢
- **STEP 2：印象的な記憶・出来事（複数可）**
  - 何歳の時、何がありましたか？（短いテキスト入力）
- **STEP 3：その時の感情と行動**
  - 感情を選択（例: 怒り、悲しみ、自己否定、諦め などをタグタップで複数選択）
  - あなたはどう行動しましたか？（例: 逃げた、我慢した、戦った）
- **STEP 4：影響人物（支配構造の特定）**
  - あなたに強い影響を与えた、コントロールしようとした人は誰ですか？（親、上司、配偶者など択一または記述）

※長文を打たせるのではなく、**「選択肢（タグ）」を多く用意**し、スマホでぽんぽんタップして進めるUXにします。

## 5. CTA配置
しつこすぎないが、決心した時にすぐ押せる配置。

1. **ファーストビュー直下**（すでに購入を決めているリピーター/紹介用）
2. **構造説明（新たな理論）の直後**（気づきを得たタイミング）
3. **利用者の声・診断質問の直後**（「自分も変わりたい」と思ったタイミング）
4. **ページ最下部**（情報を取り込み終わった最終判断の場所）
5. **追従ボタン（フローティング）**（画面下部に静かに固定配置。スクロールの邪魔にならないミニマルなもの）

## 6. データ保存・システム構成
ユーザーの手元に自動でPDFが届くまでのバックエンド処理の設計です。

1. **決済連動**: LPからStripe等の決済プラットフォームへ遷移。完了時のWebhook等でセッションを発行。
2. **フォーム入力・データ一時保持**: 決済済みユーザー専用のフォーム画面（React/Vue等）で入力を受け付け。状態（State）としてフロントに保持。
3. **バックエンド・AI解析**: 入力データをJSONでサーバー（Node.js / Python FastAPIなど）へ送信。AI APIへプロンプトを投げて解析結果テキストを生成。
4. **PDF生成・データ保存**: 解析結果をPDFファイルに変換し、クラウドストレージ（AWS S3 や Cloud Storage等）へ保存。ユーザーのメールアドレスとPDF URLをDB（PostgreSQL/Firestore等）に永続化。
5. **メール送信・ダウンロード**: 
   - 画面上にダウンロードボタンを表示。
   - 同時にSendGridなどのメール配信API経由で「人生構造レポート完成のお知らせ」とダウンロードリンクをメール送信。

## 7. URL構造
シンプルで洗練されたルーティング設計。

- `/` (Root) ＝ LPトップページ
- `/checkout` ＝ 決済プラットフォームへのリダイレクトエンドポイント
- `/q` または `/input` ＝ 決済完了後の入力フォーム画面（トークンによる認証付き）
- `/generating` ＝ AI解析中のローディング画面（UX向上のためのアニメーション配置）
- `/report` ＝ 解析結果完了・PDFダウンロード画面
- `/legal` ＝ 特定商取引法に基づく表記
- `/privacy` ＝ プライバシーポリシー
