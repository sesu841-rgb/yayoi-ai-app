# 人生構造解析レポート 公式サイト設計書 (MVP・シンプル実装版)

## 1. コンセプト・基本方針
- **名称（仮）**：人生構造解析レポート
- **コンセプト**：40年の人生を、感情ではなく構造で読む。具体・構造・因果で人生パターンを抽出する。自己分析や占いではない。
- **トーン設計**：
  - シンプル・白基調・余白多め
  - 感情的デザイン禁止・落ち着いた信頼感
  - コピー方針：煽らない、スピリチュアル禁止、感想的表現禁止、「構造」「因果」「パターン」を軸にする
- **禁止事項**：ポエム表現、占い的演出、過度な煽り、派手な装飾、感情操作型コピー

## 2. 開発・技術方針（初心者が挫折しないための全体設計）

ReactやViteといった複雑な環境構築は行わず、**「素のHTML / CSS / Vanilla JavaScript」** だけで構築します。
技術的ハードルを下げるため、以下の機能を削り（または代替し）、まずは確実に動くものを目指します。

### ❌ v1（初期版）で「削る・後回しにする」機能
1. **複雑なSPA（シングルページアプリケーション）構成**
   - 理由：画面遷移の制御や環境構築に専門知識が必要なため。
   - 代案：`index.html` の中で、JavaScriptを使って「TOP画面」「入力画面」「レポート画面」の表示/非表示（`display: none;`）を切り替えるシンプルな方式（擬似SPA）にするか、ページを分割します。
2. **レポートのPDF自動生成機能（専用ライブラリの使用）**
   - 理由：`jsPDF`などのライブラリは日本語の文字化けやレイアウト調整が非常に難しいため。
   - 代案：ブラウザ標準の「印刷機能（Ctrl+P / Command+P）」を使ってPDF保存してもらいます。「印刷される時だけ綺麗に見えるCSS（`@media print`）」を数行書くだけで実装できます。
3. **複雑な「5年ごとのステップフォーム」と自動プログレスバー**
   - 理由：「戻る・次へ」の制御や、現在のページ状態の管理が複雑になるため。
   - 代案：縦に40年分が並ぶ、シンプルなロングフォームにします。ただし、40年分を一気に入力するのは危険なため、「ブラウザに一時保存（LocalStorage）」するシンプルなボタンだけは必須として実装します。登録やログイン機能は不要です。

## 3. ページ（画面）構成

### ① TOP画面
- **FV**：「あなたの人生は偶然ではない。」＋「▶ 解析を始める」ボタン
- **対象者**：「同じ人間関係を繰り返している」など箇条書き
- **AIが行うこと**：繰り返し構造の抽出など、構造的な説明

### ② 自分史入力画面
- **UI**：1年単位の入力枠が縦に並ぶシンプルなフォーム。
  - [年齢]
  - [出来事]（事実のみ）※テキストエリア
  - [感情/状態] ※テキストエリア
  - [選択] ※テキストエリア
- **機能**：
  - 画面上部に常に追従する「入力データをブラウザに一時保存」ボタン。
  - 「解析を始める」送信ボタン。
  ※送信時、JavaScriptで「入力漏れがないか」だけを簡易チェックします。

### ③ 解析中画面（ローディング）
- **UI**：シンプルなスピナー（くるくる回るアイコン）と、「人生の構造を読み解いています...」というテキストのみ。空白の時間を耐えさせる演出です。

### ④ 構造レポート出力画面
- AIから返ってきた1500文字以上のレポートを表示する画面。
- **UI**：白背景に1カラムのクリーンなレイアウト。
- **機能**：「この画面をPDFとして保存（=ブラウザの印刷ダイアログを呼び出すだけのボタン）」

### ⑤ 伴走プログラム導線（レポート末尾）
- レポートの最下部に静かに配置。
- 「この構造を書き換えるには一人では難しい場合があります。」
- 「▶ 個別解析セッションを受ける」「▶ 90日伴走プログラムを見る」へのリンク（外部サイトの予約フォームやLPへ飛ばすだけ）。

## 4. はじめての実装ステップ（これなら着実にできます）

**ステップ1：HTMLとCSSだけで「見た目」を作る（AI連携はまだしない）**
1. フォルダを作り、`index.html` と `style.css` を作成する。
2. TOP画面、入力フォーム画面、レポート画面のHTMLをすべて書く。
3. CSSで「白基調・余白多め」の落ち着いたデザインに整える。

**ステップ2：JavaScriptで「画面の切り替え」と「一時保存」を作る**
1. `script.js` を作成する。
2. 「解析を始める」ボタンを押したらTOP画面を隠し、フォーム画面を表示する処理を書く。
3. フォームの「一時保存」ボタンを押したら、入力中のテキストを記憶する（LocalStorage）処理を書く。

**ステップ3：AIとつなぐ（ここが最後の壁）**
1. フォームの「送信」ボタンを押したら、入力されたテキストを1つの文章にまとめる処理を書く。
2. その文章を、OpenAIなどのAPIへ送信する。
3. 返ってきたレポート結果を、レポート画面のテキスト部分に流し込む。

## 5. セキュリティと公開時の安全対策（絶対に守るべき約束）

このサイトは「個人の詳細な人生」というセンシティブなデータを扱い、かつ「AI（API）」と通信を行うため、以下のセキュリティ対策を必ず設計に組み込みます。初心者でも安全にネット公開できる仕組みにします。

### 🚨 1. APIキーの絶対死守（ハードコード禁止）
初心者が最もやりがちな致命的ミスは、`script.js`（JavaScriptファイル）の中に直接OpenAI等のAPIキー（`sk-xxx...`）を書き込んでしまうことです。これをそのままネット（GitHub PagesやVercelなど）に公開すると、数分でキーが盗まれ、高額な請求を受ける恐れがあります。

- **対策：**
  JavaScript（フロントエンド）から直接AIのAPIを叩かない。
  今回のような静的サイトの場合は、以下のいずれかの方法を取ります。
  - **方法A（ユーザー入力制）**：画面に「APIキー入力欄」を作り、ユーザー自身にキーを入力してもらい、それをブラウザの一時記憶（LocalStorage）にだけ保存して通信する。※自作ツールとして自分が使うだけなら最も簡単。
  - **方法B（サーバーレス関数）**：Cloudflare Workers、Vercel Functions、Supabase Edge Functions など「APIを隠すための中継地点」だけを別に作る。フロントエンドからはそこへデータを送り、中継地点が秘密裏にAPIキーを付与してAIと通信する。

初学者の場合、まずは自作ツールとして動かすために**「方法A（自分のAPIキーを画面で入力する方式）」**から実装を始めるのが、一番安全で確実です。

### 🛡 2. 入力データの保護（サーバーに保存しない）
「過去40年の生々しい出来事」がどこかに保存されているとユーザーは不安になります。

- **対策：**
  - 万が一の漏洩を防ぐため、**データベース（DB）は一切用意しません。**
  - 入力データはユーザーの「ブラウザの中（LocalStorage）」だけに保存します。
  - AIに送ったデータを取り終わったら、レポート表示画面で「入力データを消去する」機能を持たせ、完全に手元で消えることを明記します。

### 🔒 3. 通信の暗号化（HTTPS）
万が一、カフェのWi-Fiなどでデータを見られないようにします。
- **対策：**
  - ネット公開する時は、必ず `https://` から始まるURLにします（GitHub Pages, Vercel, Netlifyなどの無料サービスを使えば自動でHTTPS化されます）。
